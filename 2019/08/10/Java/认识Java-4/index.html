interface接口

接口

接口是一种引用的数据类型，最重要的内容就是其中的抽象方法

创建格式

换了关键字interface之后，编译生成的字节码文件仍然是.java –> .class

  • java 7开始 ,接口中可以包含的内容有：
    1. 常量
    2. 抽象方法
  • java 8 开始，那么可以包含的还有：
    1. 默认方法
    2. 静态方法
  • java 9 开始：还能有
    1. 私有方法

接口的使用

抽象方法

注意：

  1. 接口不能直接使用，必须有一个实现类来实现该接口
  2. 接口的实现类必须覆盖重写接口的所有抽象方法
  3. 实现类一般起名字在最后加上Impl
  4. 接口的抽象方法的关键字必须是固定的两个public void,但是他们可以省略

接口和实现类继承时使用implements实现

默认方法

java8开始就可以在接口中定义默认方法

目的是为了解决接口升级的问题，在投入使用后，新填入抽象类会报错，这时候可以使用默认的接口方法

静态方法

java8开始

同样，接口的静态方法也是属于这个接口的，调用时直接使用接口名调用即可

私有方法

java9出现的接口的私有方法

是为了简化接口中默认方法的重复部分

常量

接口中可以定义成员变量，但是必须使用public static final三个修饰符来修饰（可以省略）

注意：

  1. final关键字表示不可以改变
  2. 必须要有初始值
  3. 常量一般要完全大写，而且多单词要用下划线进行分隔

同样常量属于接口

继承中实现类和接口的问题

注意：

  1. 接口不可以有静态代码块
  2. 接口不能有构造方法
  3. 一个类的直接父类是唯一的，但是一个类可以实现多个接口
  1. 如果实现类实现的多个接口存在方法同名的情况，只需要覆盖重写一次即可
  2. 如果没有覆盖重写所有的抽象方法，那么本类也只能是一个抽象类
  3. 如果多个接口存在重复的默认方法，实现类也要对其覆盖重写
  4. 一个类如果直接父类和接口当中的方法产生了冲突，会优先使用父类当中的方法

继承中的接口和接口的问题

  1. 接口之间可以多继承
  2. 多个父接口抽象方法同名没有关系
  3. 多个父接口的默认方法同名必须进行默认方法的覆盖重写
