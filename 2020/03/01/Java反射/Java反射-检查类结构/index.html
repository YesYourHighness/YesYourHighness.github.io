使用反射类

我们使用反射类主要就就是为了检查类的结构

一个类主要由三个部分组成

  • Field 字段域
  • Method 方法
  • Constructor 构造器

Field

主要有四个方法，以Filed为例

Class类中的 getFields、 getMethods 和 getConstructors方法将分别返回类提供的 public域、 方法和构造器数组， 其中包括超类的公有成员

Class 类的 getDeclareFields、 getDeclareMethods 和 getDeclaredConstructors方法将分别返回类中声明的全部域、 方法和构造器， 其中包括私有和受保护成员，但不包括超类的成员

区别就是：

  1. getField()只能返回public的字段、包括继承自父类的字段
  2. getDeclareField()返回全部字段，不会返回继承的字段

这是因为正常情况下，private标记的字段是不能被其他类访问的

修复错误我们可以调用setAccessible方法，设置后，我们就可以访问任意类型了

setAccessible(true)可能会失败。如果JVM运行期存在SecurityManager，那么它会根据规则进行检查，有可能阻止setAccessible(true)。例如，某个SecurityManager可能不允许对java和javax开头的package的类调用setAccessible(true)，这样可以保证JVM核心库的安全。

可以访问了，当然就可以赋值了
